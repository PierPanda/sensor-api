Sensor API

W3C Generic Sensor API & présentation & demo

C'est quoi ?

Une API qui permet à une page web de lire les capteurs physiques de l'appareil :

  1. Accéléromètre détecte les mouvements
  2. Gyroscope détecte les rotations
  3. Magnétomètre détecte le nord magnétique
  4. Lumière ambiante mesure la luminosité
  5. et d'autres...
Avant, chaque capteur avait sa propre syntaxe. Maintenant, un seul modèle pour tous.

Un peu d'histoire

De l'iPhone à une API standardisée en 10 ans :

AnnéeCe qui s'est passé

2009–11Premiers essais : DeviceMotionEvent ça marche, mais chaque navigateur fait différemment.
2015Le W3C lance un groupe de travail pour créer un vrai standard.

Aujourd'huiFonctionne sur Chrome et Edge. Firefox et Safari ne l'ont pas encore.

Pourquoi ? Le problème

Avant, chaque capteur avait sa propre syntaxe. Exemple avec l'accéléromètre :

      // Ancienne méthode pas de contrôle 
        window.addEventListener(
          'devicemotion', 
          (event) => {
            console.log(event.accelerationIncludingGravity.x);
          }
        );

Les problèmes :

  • Impossible de choisir combien de fois par seconde lire le capteur
  • Pas de demande de permission à l'utilisateur
  • Pas de timing précis pour synchroniser plusieurs capteurs
  • Chaque navigateur faisait à sa façon

Pourquoi ? La solution

Avec la Generic Sensor API, tous les capteurs fonctionnent pareil :

  1. Créer le capteur en précisant la fréquence voulue (ici 60 fois/seconde)
  2. Écouter une fonction appelée à chaque nouvelle mesure
  3. Démarrer le capteur ne s'active que quand on le demande
const sensor = new Accelerometer({ frequency: 60 });

sensor.addEventListener('reading', () => {
  console.log(sensor.x, sensor.y, sensor.z); // en m/s²
});

sensor.start();

Sécurité & permissions

L'API protège l'utilisateur :

  • Site en HTTPS obligatoire (ou localhost pour tester)
  • Le navigateur demande la permission avant d'activer un capteur
  • Les iframes doivent être autorisées explicitement par le serveur
// Vérifier si l'utilisateur a donné la permission
const result = await navigator.permissions.query({ name: 'accelerometer' });
// result.state → 'granted' | 'denied' | 'prompt'

Cycle de vie d'un capteur

const sensor = new Gyroscope({ frequency: 60 });

sensor.addEventListener('activate', () => console.log('prêt'));
sensor.addEventListener('error', (e) => console.error(e.error));
sensor.addEventListener('reading', () => {
  console.log(sensor.x, sensor.y, sensor.z);
});

sensor.start(); // démarre le capteur
sensor.stop();  // arrête et libère la batterie
⚠ Un capteur actif consomme de la batterie. Toujours appeler .stop() quand on n'en a plus besoin.
